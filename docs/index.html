gh repo fork Grumpified-OGGVCT/Sovereign_Agentic_OS_with_HLF --clone

🔑 Step 2 — Configure API Keys & Services

Copy the template and fill in your credentials:

cp .env.example .env

Required keys in your .env file:

  • REDIS_PASSWORD — Redis auth (for gateway message bus)
  • OPENROUTER_API_KEY — Cloud model access (Sovereign tier)
  • GITHUB_TOKEN — For Copilot factory / agent dispatch

Optional but recommended:

  • OLLAMA_HOST — Local model endpoint (default: http://localhost:11434)
  • SLACK_WEBHOOK_URL — Agent alert notifications
  • STREAMLIT_SECRET_KEY — GUI session encryption

⚠️ Never commit your .env file — it's already gitignored with 70+ rules.

Save all keys in .env at the project root. The system auto-loads them via python-dotenv.

⚡ MoMA Model Router — Architecture

This is how the real model routing works when running locally. It is not simulated in this static demo — it requires a live Ollama or OpenRouter backend.

🏗️ Routing Flow

1
Intent Analysis — HLF intent is parsed for complexity, domain, and latency constraints
2
Tier Resolution — Deployment tier (Hearth/Forge/Sovereign) determines available backends
3
Model Selection — MoMA Router picks optimal model based on: task type, VRAM budget, latency SLA, and benchmark scores
4
Dispatch — Request routed to Ollama (local) or OpenRouter (cloud) with gas metering
5
Result + Metrics — Response captured with real latency/throughput, logged to ALIGN Ledger

📊 Deployment Tiers

🏠 Hearth

Local Ollama only
qwen2.5:7b, phi3:mini
Max latency: 500ms

🔥 Forge

Ollama + OpenRouter fallback
codestral, deepseek-coder
Max latency: 2000ms

👑 Sovereign

Full cloud access
claude-3.5, gpt-4o
No latency limit

🏗️ How It Works vs Traditional RAG

Dimension Traditional RAG Infinite RAG
Context Limit 4K–128K tokens (hard ceiling) Unlimited — fractal compression
Memory Stateless per session Persistent — fact_store survives reboots
Retrieval Speed 10-100ms (vector search) <1ms — Redis hot cache pre-loads topics
Deduplication None (embed same text repeatedly) SHA-256 cache eliminates redundant embeds
Self-Improvement Static — needs human re-indexing Dream State — nightly auto-compression & rule synthesis
Governance No write controls ALIGN Ledger enforces all memory writes
